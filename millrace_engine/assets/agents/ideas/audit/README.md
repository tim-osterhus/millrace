# Audit Intake

Use this directory family for audit work items only:
- `agents/ideas/audit/incoming/`
- `agents/ideas/audit/working/`
- `agents/ideas/audit/passed/`
- `agents/ideas/audit/failed/`

All audit work items should follow:
- `agents/specs/templates/audit_template.md`

At minimum each ticket must retain:
- stable `audit_id`
- explicit `scope`
- explicit `trigger`
- `status` that matches queue location
- clear PASS/FAIL decision with evidence pointers
